## About the book

Most people learn Linux by accident: a command copied from a forum, a service that finally started, a config file that mysteriously worked. *The Linux Craftsman* replaces that accidental knowledge with genuine understanding. It is a book about why the system behaves the way it does, so that the next unfamiliar problem becomes solvable rather than terrifying.

The book moves from the surface to the foundations. It begins with real command-line fluency, the kind that turns a ten-minute task into a ten-second one, then descends through process management, init systems, and finally the kernel primitives that make modern containers and isolation possible. Along the way it treats the shell, `systemd`, and the `/proc` filesystem as the coherent, learnable system they actually are.

This is a hands-on book. You are expected to open a terminal, break things in a safe environment, and watch the system respond. Understanding follows from doing.

## What's inside

1. Shell fluency: pipes, redirection, job control, and writing scripts you can trust
2. Files, permissions, and the filesystem hierarchy explained from first principles
3. Processes, signals, and what really happens when a program runs
4. Service management with `systemd`, units, and journald
5. Namespaces and cgroups: the building blocks beneath every container
6. Networking from the interface up, including troubleshooting that actually works
7. Observability and debugging with the tools already on every box

## Who it's for

This book is for developers who want to stop fearing the server, for system administrators who want to understand rather than memorize, and for anyone moving toward platform, DevOps, or SRE work. If you can navigate a directory and edit a file, you have enough to begin.

You will get the most out of it if you have access to a Linux machine or virtual machine you can experiment with freely. Each chapter assumes you would rather see a primitive in action than read its abstract definition.

By the final chapter, the system that once felt like a black box should feel like a workshop: a set of sharp, well-understood tools that do exactly what you tell them, and that you can reason about with confidence.
